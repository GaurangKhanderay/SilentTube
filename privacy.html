SilentTube — Privacy Policy

Zero Data Collection

Last updated: April 2026

What data does SilentTube collect?

None. Zero. Absolutely nothing.

SilentTube does not collect, transmit, store on any server, or share any personal data, browsing data, or user information of any kind.

What data is stored locally?

SilentTube saves your toggle preferences (on/off for each feature) using Chrome's built-in chrome.storage.sync API. This data is stored entirely on your device. If Chrome Sync is enabled in your browser, these settings may sync across your own devices via your Google account — but they are never transmitted to any SilentTube server (there is no server).

The only data ever stored:

  • hideShorts: true/false
  • hideRecommended: true/false
  • hideHomeFeed: true/false
  • hideEndCards: true/false
  • hideAutoplay: true/false
  • hideComments: true/false
  • hideTrending: true/false

Does SilentTube make any network requests?

No. SilentTube contains zero fetch(), XMLHttpRequest(), or WebSocket() calls. It cannot and does not communicate with any remote server. This is enforced by the extension's Content Security Policy.

Does SilentTube use analytics or tracking?

No. There are no analytics libraries, no crash reporting services, no A/B testing frameworks, and no third-party SDKs of any kind. The extension has zero external dependencies.

Does SilentTube read YouTube page content?

SilentTube's content script runs on YouTube pages and injects a <style> tag to hide certain UI elements using CSS. It does not read, parse, or extract any content from the YouTube page (such as video titles, watch history, or account data).

Open Source

SilentTube's source code is fully auditable. Every line of code is contained in the extension package you installed. There are no obfuscated or minified files.
